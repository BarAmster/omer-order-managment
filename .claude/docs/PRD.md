## 1. רקע

עסק לאספקת בטון (העסק של עומר) מנהל הזמנות, לקוחות ומחירים בצורה ידנית
(טלפון/וואטסאפ/דפים/אקסל). אין מקור אמת אחד, קשה לראות לוח הזמנות, וקשה לדעת
רווחיות לכל עבודה. האפליקציה מרכזת את כל אלה במערכת אחת נוחה לנייד.

---

## 2. הגדרת הבעיה

**בעיה עסקית:** ניהול הזמנות מבוזר → טעויות תמחור, חוסר נראות על הלו"ז ועל רווחיות.

**בעיה טכנית:** אין מערכת ממוחשבת ייעודית; חישובי תמחור (בטון לפי חוזק/סוג/שקיעה,
משאבה per-job) נעשים בראש/ידנית וחשופים לטעויות.

**נקודות כאב:**
- אין תמונה אחת של הזמנות יומי/שבועי/חודשי.
- תמחור בטון/משאבה מורכב ונוטה לטעויות.
- אין מעקב רווחיות (מחיר ללקוח מול עלות מהמפעל).
- אין הפקה מהירה של הצעת מחיר.

---

## 3. מטרות

**מטרות עיקריות:**
- לוח הזמנות מהיר לנייד עם תצוגות יומי/שבועי/חודשי.
- יצירת הזמנה בכמה הקלקות עם **תמחור עלות אוטומטי** מהמחירון.
- מעקב **רווחיות** לכל פריט ולכל הזמנה.
- ניהול לקוחות קבועים עם אתרים והיסטוריה.
- ניהול מחירונים פר-מפעל.
- הפקת **הצעת מחיר** מודפסת.

**מדדי הצלחה:**
- יצירת הזמנה מלאה ב-< דקה.
- 0 טעויות בחישוב עלות (העלות נגזרת אוטומטית מהמחירון).
- כל ההזמנות והלקוחות במקום אחד נגיש מהנייד.

---

## 4. Non-Goals (מחוץ לגבולות כרגע)

- ריבוי משתמשים / הרשאות / תפקידים — **משתמש יחיד בלבד**.
- פורטל/אזור ללקוחות-קצה.
- חשבוניות/הנהלת חשבונות מלאה (רק הצעת מחיר).
- אפליקציה נייטיב (זו web app נייד-first).
- אינטגרציה עם מערכות מפעל חיצוניות.

---

## 5. משתמשים / Personas

### משתמש יחיד: הבעלים (עומר / אדמין)
מנהל את כל ההיבטים: הזמנות, לקוחות, מחירונים. צריך מהירות, נוחות בנייד, ותמחור
שלא טועה. רואה נתונים פיננסיים (עלות/רווח) שמוסתרים כברירת מחדל בתוך ההזמנה.

---

## 6. User Flows

### Flow 1: יצירת הזמנה
1. לוחץ FAB (+) בעמוד ההזמנות.
2. מחפש ובוחר לקוח (autocomplete).
3. בוחר אתר (מאתרי הלקוח) או מקליד אתר חופשי.
4. בוחר מפעל, תאריך+שעה, אמצעי תשלום.
5. מוסיף פריטים (בטון/משאבה/נלווה); העלות מחושבת אוטומטית, מזין מחיר ללקוח.
6. שומר.

**תנאי קבלה:** חובה לקוח + מפעל + אתר + תאריך + לפחות פריט אחד. עלות בטון/משאבה
תואמת את המחירון. כמות משאבה = כמות בטון.

### Flow 2: ניהול ורווחיות הזמנה
1. לוחץ על הזמנה → `OrderDrawer`.
2. רואה פירוט, סה"כ, ומרחיב "מצב כספי" לראות רווח לפריט וכולל.
3. משנה סטטוס (כולל דחייה לתאריך), מפיק הצעת מחיר, עורך או מוחק.

### Flow 3: ניהול מחירון מפעל
1. עובר ל"מחירונים", פותח מפעל.
2. מעדכן מחיר בסיס בטון, תוספות פרמטרים, משאבות, מוצרים נלווים.

## 8. דרישות לא-פונקציונליות

| קטגוריה | דרישה |
|---------|-------|
| ביצועים | טעינה מהירה בנייד; שאילתות ממוקדות לטווח התאריכים. |
| נוחות | Mobile-first, RTL, מינימום הקלקות. |
| אבטחה | Supabase Auth + RLS (`authenticated` בלבד); אין חשיפת service-role ל-client. |
| תחזוקתיות | קומפוננטות קטנות, לוגיקת תמחור מרוכזת ב-`lib/pricing.js`. |
| Scalability | משתמש יחיד; אין צורך בקנה מידה גדול כרגע. |
